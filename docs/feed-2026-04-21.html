Summary

A curated set of 12 posts, papers, and projects selected for technical depth and relevance.

Papers

A Real-Time Bike-Pedestrian Safety System with Wide-Angle Perception and Evaluation Testbed for Urban Intersections

arXiv Vision | Mehmet Kerem Turkcan | 2026-04-18

  • This is one of the strongest items in the bundle.
  • It presents a real-time urban safety system built around a wide-angle fisheye camera, and the paper goes well beyond detection by detailing a calibration pipeline, a lookup-table-based ground-plane projection stage, a conformance simulation framework, and explicit deployment-risk analysis.
  • The combination of geometry, edge deployment, and testbench thinking makes it far more useful than a generic perception paper.
  • Distinctive point: It couples fisheye calibration, closed-form projection, and deployment gating in a single end-to-end safety system.

Fringe Projection Based Vision Pipeline for Autonomous Hard Drive Disassembly

arXiv Vision | Badrinath Balasubramaniam | 2026-04-19

  • This paper is a strong engineering piece because it uses structured-light style sensing through fringe projection and then integrates it with segmentation and depth completion for robotic disassembly.
  • The pixel-wise alignment argument is especially compelling: the same camera-projector system supports both depth and semantic localization, which simplifies downstream 3D reasoning in a way many RGB-D pipelines do not.
  • Distinctive point: It combines fringe projection, depth completion, and segmentation in one inherently aligned 3D pipeline.

Keep It CALM: Toward Calibration-Free Kilometer-Level SLAM with Visual Geometry Foundation Models via an Assistant Eye

arXiv Robotics and Geometry | Tianjun Zhang | 2026-04-16

  • CAL2M targets large-scale SLAM with visual geometry foundation models and directly addresses intrinsic correction, pose correction, and geometric distortion.
  • The paper is especially relevant because it treats calibration and geometry problems as deployment bottlenecks rather than just benchmark details.
  • Even the unusual assistant-eye idea is tied to a concrete systems goal: reducing scale ambiguity and improving global consistency.
  • Distinctive point: It combines intrinsic correction and nonlinear map alignment in a calibration-aware SLAM framework built around geometry foundation models.

SemLT3D: Semantic-Guided Expert Distillation for Camera-only Long-Tailed 3D Object Detection

arXiv Vision | Hao Vo | 2026-04-20

  • SemLT3D is a solid camera-only 3D detection paper with a concrete angle on long-tail classes, which matters in safety-critical driving scenarios where rare categories can still be important.
  • The semantic-guided expert routing and CLIP-informed distillation make it more interesting than a generic detector-improvement paper.
  • Distinctive point: It attacks long-tail camera-only 3D detection using semantic expert routing and semantic projection distillation.

CooperDrive: Enhancing Driving Decisions Through Cooperative Perception

arXiv Vision | Deyuan Qu | 2026-04-15

  • CooperDrive is a strong automotive-perception paper centered on cooperative perception and BEV-based decision support under occlusion and non-line-of-sight conditions.
  • It is more planning-facing than camera-model work, but still highly relevant because it sits inside the real driving stack rather than isolated detector benchmarks.
  • The object-level sharing and fusion angle makes it a good complement to the camera-only perception items already in the issue.
  • Distinctive point: It links cooperative perception directly to driving decisions instead of treating perception as a standalone module.

Projects

eowjd0512/fisheye-calib-adapter

GitHub Vision Geometry C++ | eowjd0512 | 2026-04-07

  • This repository is unusually relevant if you care about real camera models rather than generic OpenCV calibration scripts.
  • It focuses on converting and reconciling multiple fisheye models, including double-sphere, Kannala-Brandt, EUCM, UCM, Kalibr, and WoodScape-style formulations, which is exactly the kind of glue code that shows up in real multi-camera and automotive perception stacks.
  • Distinctive point: It is about cross-model fisheye calibration interoperability, not just running one calibration routine.

xixu-me/avm

GitHub Fisheye Geometry | xixu-me | 2026-04-20

  • A practical surround-view project built around fisheye cameras, OpenCV, and bird's-eye-view generation.
  • It is directly relevant to automotive camera geometry, calibration, and projection workflows rather than generic vision demos.
  • It is useful as a concrete implementation reference when you care about how multi-camera AVM systems are actually assembled.
  • Distinctive point: It connects fisheye imagery to a full around-view monitoring pipeline instead of stopping at isolated calibration steps.

fundamentalvision/BEVFormer

GitHub Automotive Perception | fundamentalvision | 2026-04-21

  • BEVFormer is already an important reference point for camera-only autonomous-driving perception, and it remains highly relevant if you are interested in BEV representations and multi-camera reasoning.
  • Even though it is not new, it is one of the few GitHub items in the bundle that clearly belongs in an advanced automotive-vision digest.
  • Distinctive point: It is a canonical camera-only BEV perception framework rather than a small demo repository.

sourishg/fisheye-stereo-calibration

GitHub Vision Geometry C++ | sourishg | 2026-03-06

  • Stereo fisheye calibration is harder and more practically relevant than monocular calibration for many robotics and surround-view systems, so this repository earns a place on depth alone.
  • It gives you a concrete C++ and OpenCV reference point for handling stereo geometry under fisheye distortion, which is exactly where many toy calibration guides stop being useful.
  • Distinctive point: It tackles stereo fisheye calibration directly, which is much closer to real multi-camera systems.

astar-ai/calicammono

GitHub Vision Geometry C++ | astar-ai | 2026-03-16

  • CaliCamMono looks like a practical tool for rectifying fisheye imagery into multiple output parameterizations such as cylindrical, perspective, and longitude-latitude projections.
  • That makes it useful as a working reference for the downstream side of calibration, where the question is not only estimating parameters but also choosing how to remap the imagery for later perception tasks.
  • Distinctive point: It emphasizes multiple fisheye rectification targets rather than only calibration parameter estimation.

Posts

Oct 24, 2025 Narrow FOV Calibration Made Easy(er)

Tangram Vision Blog | 2026-04-21

  • This is one of the clearest matches for your interests in the entire bundle.
  • Narrow-FOV calibration is a real engineering problem that regularly breaks downstream extrinsics and stereo systems even when reprojection error looks acceptable, and Tangram is writing directly about why that happens and how to handle it.
  • That is exactly the right combination of theory and applied calibration pain.
  • Distinctive point: It focuses on the failure modes of narrow-FOV calibration rather than generic calibration basics.

Dec 9, 2025 Plane Fitting, Part 2: Manifold Optimization, without Lie Groups

Tangram Vision Blog | 2026-04-21

  • This is one of the best blog picks because it is directly about geometry and optimization rather than product promotion.
  • Plane fitting is a deceptively deep topic in perception pipelines, and a post about manifold optimization without leaning on Lie-group boilerplate sounds exactly like the kind of mathematically grounded engineering note worth keeping.
  • Distinctive point: It revisits plane fitting through manifold optimization instead of the usual cookbook treatment.
